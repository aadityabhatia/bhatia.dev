Software Engineering with Agentic AI

Build software without drowning in prompts!

Thursday, July 23, 2026

The Problem

  • Syntax is now a commodity
    • AI produces content instantly
  • The “Black Box” Trap
    • Unstructured prompting (“vibe coding”) bypasses systems thinking and domain expertise
    • Students lack architectural understanding
  • Brittle Outcomes
    • Trial-and-error prompting does not scale to complex, reliable software

How do we transition from unstructured prompting to disciplined, agentic systems engineering?

Core Concepts

Harness

  • Frameworks that manage the control flow and context for agents
  • A step beyond basic prompting and chatting
  • Makes your AI agents agentic!

AGENTS.md

  • README for agents
  • Context provided to every agent operating in the directory
  • Nest AGENTS.md files for precise context

Tools

  • Functions for agents to call
  • Allow agents to interact with systems and APIs
  • Typically provided by the harness

Skills

  • Reusable instructions for AI agents
  • Agents can load skills as needed
  • Typically add 5k-20k tokens of context
  • Adds new capabilities; bloats context window

The Big Idea

  • Develop a proposal. Produce specifications.
  • Write the design / architecture document.
  • Review and refine the plan. Implement it.
  • Test the implementation. Develop a plan to fix bugs. Implement it.

Measure of Performance

  • Token maxxing!!

    • Throwing tokens at problems until they go away.
    • Brute forcing was rewarded by tech companies; employees spending below threshold were penalized
    • Tokenmaxxing: great for AI vendors

  • Value maxxing: Maximizing output quality per token

  • Model maxxing: Efficient Model Routing

    • Frontier models develop plans and work with ambiguous requirements.
    • Smaller, faster models perform simpler, well-defined tasks.

  • What is a good measure of effectiveness?

Guiding Principles

  1. Markdown everywhere.
  2. Own your prompts.
  3. Context window.
  4. Control flow: deterministic process to manage agent interactions.
  5. Agents: small, simple, and focused. Each agent does one thing well.

## Markdown: The Lingua Franca

- Beautiful, simple, and human-readable.
    - `#` and `##` for headings.
    - `-` for lists.
    - `1.` and `1.` and `1.` for numbered lists.
    - bold and _italic_ for emphasis.
    - single backtick for inline code and three backticks for code blocks
    - `>` for blockquotes.
    - `---` for horizontal rules.
    - [Link](https://example.com) for hyperlinks.
    - ![description](image.png) for images.

- Human-readable + machine-parsable = ✨

Control Flow

  • Ideally, deterministic, predictable, and testable.
  • Realistically, a more capable agent manages the control flow and tasks simpler sub-agents.
  • Use intermediate artifacts to pause execution, allow human review, and ensure reliable outcomes.
  • LLM-as-a-judge: Use LLMs to review and approve artifacts.
  • Open-ended prompting is used to refine the plan.
  • Implementation is driven by artifacts, not prompts.

Context Window

  • Context is precious; don’t dump the entire repository!
  • The “Need to Know” Basis: Feed agents only the specific files or functions they need.
  • Shorter context = faster responses, cheaper calls, and fewer hallucinations.
  • Skills often bloat context windows. Use them judiciously.
    • Instead, use custom agents that will do one thing well.

Prompts

  • Treat prompts as Code.
  • Version control your system instructions (Git is your friend).
  • Separate instructions from data using clear templating.
  • Modularity: Keep prompts scoped to a single, testable objective.

Simple, Focused Agents

  • The Unix Philosophy for AI: Do one thing and do it well.
  • Break large tasks into specialized, easily debuggable roles:
    • 🧠 Planner Agent: Drafts the implementation steps.
    • 💻 Coder Agent: Writes the targeted code.
    • 🕵️ Reviewer Agent: Checks for bugs and anti-patterns.
  • Modular agents let you use smaller, faster models for simpler tasks (Model maxxing!).
